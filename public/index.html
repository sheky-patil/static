50+ Interview Q&As with model answers

Every chapter ends with real interview questions and answers written to demonstrate senior-level thinking.

Q: How do you handle cache invalidation — the "hardest problem in CS"?
A: Three strategies: (1) TTL-based expiry — simplest, acceptable staleness. (2) Event-driven invalidation — when a DB record updates, publish an event that deletes the cache key. (3) Write-through — update cache synchronously on every write. In interviews say: "I'd use TTL as the safety net and event-driven invalidation for high-consistency requirements."
Q: How does Kafka guarantee message ordering?
A: Kafka guarantees ordering WITHIN a partition, not across partitions. All messages with the same partition key (e.g., user_id) go to the same partition, ensuring ordered processing per user. Different users' events can be processed in parallel across partitions. This is why partition key selection is critical in system design.
Q: When would you choose Cassandra over PostgreSQL?
A: Cassandra when you need massive write throughput with time-series or append-only data — e.g., IoT sensor data, activity logs, message history. Cassandra is optimised for writes (append-only LSM tree) and can handle millions of writes/sec across many nodes. PostgreSQL is superior for complex queries, joins, and ACID transactions.
